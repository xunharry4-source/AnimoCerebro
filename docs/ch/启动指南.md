## 快速开始

### 一键启动（推荐）

```bash
# 首次使用：初始化环境
./scripts/setup_env.sh

# 启动开发环境
make dev
```

访问：
- 前端：http://127.0.0.1:5173
- 后端：http://127.0.0.1:8000
- API 文档：http://127.0.0.1:8000/docs

### 遇到问题？

```bash
# 一键清理并重启
make restart-dev
```

## 环境准备

### 前置要求

- **Python**: 3.10+
- **Node.js**: 18+
- **npm**: 9+
- **操作系统**: macOS, Linux, Windows (WSL)

### 检查环境

```bash
# 检查 Python 版本
python3 --version

# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version
```

### 安装依赖

#### 方法 1: 一键安装（推荐）

```bash
./scripts/setup_env.sh
```

该脚本会：
1. 创建 Python 虚拟环境 `.venv`
2. 安装 Python 依赖（requirements.txt + requirements-dev.txt）
3. 安装前端依赖（npm install）

#### 方法 2: 分步安装

```bash
# 1. 安装后端依赖
make backend-install

# 2. 安装前端依赖
make frontend-install
```

#### 方法 3: 手动安装

```bash
# 后端
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt -r requirements-dev.txt

# 前端
cd src/admin-portal
npm install
```

### 配置环境变量

```bash
# 复制示例配置
cp .env.example .env

# 编辑配置文件，填入 API Key
vim .env
```

最少需要配置一个 LLM Provider：

```env
# 选项 1: 使用本地网关（推荐开发）
OPENAI_COMPAT_BASE_URL=http://localhost:8317/v1
OPENAI_COMPAT_API_KEY=your-api-key-1

# 选项 2: 直接使用 OpenAI
OPENAI_API_KEY=sk-...

# 选项 3: 使用 Gemini
GEMINI_API_KEY=AIza...
```

## 启动方式

### 1. 一键启动（make dev）

**适用场景**: 日常开发

```bash
make dev
```

**等价命令**:
```bash
./scripts/dev_all.sh
```

**特点**:
- ✅ 自动检查依赖
- ✅ 自动检查端口占用
- ✅ 后端健康检查
- ✅ 同时启动前后端
- ✅ 支持热重载

**输出示例**:
```
Starting Zentex web console
Starting backend on http://127.0.0.1:8000
Using WebSocket implementation: websockets-sansio
Waiting for backend health (max 30 seconds)...
Backend is ready after 3 seconds
Starting frontend on http://127.0.0.1:5173
Web console is starting:
  Frontend: http://127.0.0.1:5173
  Backend:  http://127.0.0.1:8000
  API:      http://127.0.0.1:8000/api/web/plugins/cognitive
```

### 2. 一键重启（make restart-dev）

**适用场景**: 
- 端口被占用
- 进程卡死
- 修改了核心配置

```bash
make restart-dev
```

**等价命令**:
```bash
./scripts/restart_dev.sh
```

**特点**:
- ✅ 清理占用端口的旧进程
- ✅ 清理大型运行时文件（>1MB 的 .jsonl）
- ✅ 完全重启前后端
- ✅ 确保干净启动

**重启流程**:
1. 查找占用端口的进程
2. 尝试优雅关闭（TERM 信号）
3. 强制关闭（KILL 信号，最多 5 次尝试）
4. 清理大型日志文件
5. 调用 `make dev` 重新启动

### 3. 单独重启后端

**适用场景**: 只修改了后端代码

```bash
./scripts/restart_backend.sh
```

**特点**:
- ✅ 仅重启后端
- ✅ 保留前端运行
- ✅ 清理后端进程和日志

### 4. 单独重启前端

**适用场景**: 只修改了前端代码

```bash
./scripts/restart_frontend.sh
```

**特点**:
- ✅ 仅重启前端
- ✅ 保留后端运行
- ✅ 清理前端缓存

### 5. 仅启动前端

**适用场景**: 后端已在运行，只需调试前端

```bash
make frontend-dev
```

**等价命令**:
```bash
cd src/admin-portal && npm run dev
```

### 6. 仅启动后端

**适用场景**: 只需要后端 API

```bash
export PYTHONPATH=src
python -m uvicorn zentex.web_console.dev_server:app --reload --ws websockets-sansio --host 127.0.0.1 --port 8000
```

### 7. 生产模式启动

**适用场景**: 部署到生产环境

```bash
# 后端
export PYTHONPATH=src
python -m uvicorn zentex.web_console.app:app --host 0.0.0.0 --port 8000 --workers 4

# 前端（先构建）
cd src/admin-portal
npm run build

# 使用静态文件服务器（如 nginx）
```

## 启动流程详解

### dev_all.sh 启动流程

```
1. 环境检查
   ├─ 检查 Python 版本
   ├─ 检查 node_modules 是否存在
   └─ 检查后端依赖（fastapi, pydantic, uvicorn, websockets）

2. 端口检查
   ├─ 检查 BACKEND_PORT (默认 8000)
   └─ 检查 FRONTEND_PORT (默认 5173)

3. 启动后端
   ├─ 设置 PYTHONPATH=src
   ├─ 启动 Uvicorn: uvicorn zentex.web_console.dev_server:app
   ├─ 参数: --reload --ws websockets-sansio --host 127.0.0.1 --port $BACKEND_PORT
   └─ 后台运行，记录 PID

4. 健康检查
   ├─ 等待后端就绪（最多 30 秒）
   ├─ 每秒检查一次: curl http://127.0.0.1:$BACKEND_PORT/api/web/overview
   ├─ 检查后端进程是否仍在运行
   └─ 若失败则中止启动

5. 启动前端
   ├─ 进入 src/admin-portal 目录
   ├─ 设置 VITE_BACKEND_PORT=$BACKEND_PORT
   └─ 启动 Vite: npm run dev -- --host 127.0.0.1 --port $FRONTEND_PORT

6. 输出访问地址
   ├─ Frontend: http://127.0.0.1:$FRONTEND_PORT
   ├─ Backend:  http://127.0.0.1:$BACKEND_PORT
   └─ API:      http://127.0.0.1:$BACKEND_PORT/api/web/plugins/cognitive

7. 等待进程结束
   └─ trap cleanup EXIT INT TERM（捕获退出信号，清理子进程）
```

### restart_dev.sh 重启流程

```
1. 读取配置
   ├─ BACKEND_PORT (默认 8000)
   ├─ FRONTEND_PORT (默认 5173)
   └─ WS_IMPLEMENTATION (默认 websockets-sansio)

2. 关闭后端
   ├─ 查找占用 BACKEND_PORT 的进程
   ├─ 发送 TERM 信号
   ├─ 等待 0.2 秒
   ├─ 若仍占用，发送 KILL 信号
   ├─ 重复最多 5 次
   └─ 若仍失败，输出错误并中止

3. 关闭前端
   ├─ 查找占用 FRONTEND_PORT 的进程
   └─ 同后端关闭流程

4. 清理残留进程
   └─ pkill -f "uvicorn zentex.web_console.dev_server:app"

5. 二次确认
   ├─ 再次检查端口状态
   └─ 若仍占用，强制清理

6. 清理大型日志文件
   ├─ 扫描 .zentex/runtime/*.jsonl
   ├─ 找出 > 1MB 的文件
   └─ 清空内容（保留文件）

7. 重新启动
   └─ 调用 make dev
```

## 故障排查

### 常见问题

#### 1. 端口被占用

**症状**:
```
Backend port already in use: 8000
Run: make restart-dev
```

**解决**:
```bash
# 方法 1: 使用重启命令
make restart-dev

# 方法 2: 手动查找并杀死进程
lsof -nP -iTCP:8000 -sTCP:LISTEN
kill -9 <PID>

# 方法 3: 切换端口
BACKEND_PORT=8001 make dev
```

#### 2. 依赖缺失

**症状**:
```
Backend dependencies missing (see requirements.txt).
Create a local virtualenv and install dependencies there:
  python3 -m venv .venv
  .venv/bin/python -m pip install -r requirements.txt -r requirements-dev.txt
```

**解决**:
```bash
make backend-install
```

#### 3. 前端依赖未安装

**症状**:
```
admin-portal dependencies are not installed.
Run: make frontend-install
```

**解决**:
```bash
make frontend-install
```

#### 4. 后端健康检查失败

**症状**:
```
Backend failed readiness check: http://127.0.0.1:8000/api/web/overview
```

**可能原因**:
- LLM API Key 未配置
- 数据库损坏
- 代码语法错误

**解决**:
```bash
# 1. 检查 .env 配置
cat .env

# 2. 查看后端日志（启动时会输出）

# 3. 手动启动后端查看详细错误
export PYTHONPATH=src
python -m uvicorn zentex.web_console.dev_server:app --reload

# 4. 清理数据库（谨慎操作）
rm -rf .zentex/kuzu_db
make restart-dev
```

#### 5. WebSocket 连接失败

**症状**: 前端无法连接后端 WebSocket

**解决**:
```bash
# 检查 WebSocket 实现
echo $ZENTEX_WS_IMPLEMENTATION

# 应该是 websockets-sansio
# 如果不是，重新安装依赖
make backend-install
make restart-dev
```

#### 6. Python 版本过低

**症状**:
```
SyntaxError: invalid syntax
```

**解决**:
```bash
# 检查 Python 版本
python3 --version

# 需要 3.10+
# 如果版本过低，升级 Python
```

#### 7. Node.js 版本过低

**症状**:
```
npm ERR! Unsupported engine
```

**解决**:
```bash
# 检查 Node.js 版本
node --version

# 需要 18+
# 如果版本过低，升级 Node.js
```

### 日志查看

#### 后端日志

后端日志会实时输出到终端：

```bash
make dev
# 日志直接显示
```

#### 前端日志

前端日志在 Vite 控制台：

```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://127.0.0.1:5173/
➜  Network: use --host to expose
```

#### Transcript 日志

运行时事件记录在 JSONL 文件中：

```bash
# 查看最近的 transcript
tail -n 20 .zentex/runtime/web_console_transcript.jsonl

# 查看所有 transcript 文件
ls -lh .zentex/runtime/*.jsonl
```

### 性能优化

#### 加速启动

```bash
# 1. 清理大型日志文件
for f in .zentex/runtime/*.jsonl; do
  if [ $(stat -f%z "$f" 2>/dev/null || echo 0) -gt 1048576 ]; then
    > "$f"
  fi
done

# 2. 使用重启命令（会自动清理）
make restart-dev
```

#### 减少内存占用

```bash
# 限制 Uvicorn workers
export UVICORN_WORKERS=1

# 禁用前端 HMR（不推荐开发时使用）
# 修改 vite.config.ts
```
