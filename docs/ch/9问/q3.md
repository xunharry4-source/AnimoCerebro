# Q3: 假设身份 / 我应该以什么身份理解本轮任务

## 核心使命
根据 Q1 环境和 Q2 自我观察，生成本轮可解释的身份/角色假设，为 Q4 目标生成提供视角。若与人类设置角色冲突，人类设置角色为最终生效角色，Q3 必须显示差异但不得覆盖。

## 详细功能规范
- **输入契约**：Q3 必须消费 Q1 和 Q2 的完整结构化分析结果。禁止消费 Q4 及以后模块的推演结果（禁止时序污染）。
- **最高优先级规则：人类设置角色优先**：如果人工配置中已经存在 `human_configured_role`，则该角色永远是本轮 `effective_role` 的权威来源。
- **差异必须可见**：当 AI 推断角色与人类设置不一致时，必须输出 `RoleConflictAssessment` / `RoleDeltaReport`，明确展示差异。
- **内部认知轨返回**：`InternalIdentityHypothesisSet`，包含有效内部角色、AI 推断候选、原因推理、身份内核引用及差异报告。
- **外部执行轨返回**：`ExternalExecutionIdentityHypothesisSet`，包含有效外部姿态、委派姿态、代表权限制及差异报告。
- **冲突处理**：
    - 无人工设置时：Q3 推荐角色仅作为候选。
    - 存在实质冲突时：必须标记 `requires_human_review=true`，但仍以人类设置为准。
- **规则**：Q3 不改写 IdentityKernel，不改变主角色，不生成目标。

## 业务示例

### 示例 A：内部认知轨 / 工程鲁棒性升级
- **本轮身份**：内部工程维护者 / 自由进化候选规划者。

### 示例 B：外部执行轨 / Reddit 公开内容研究
- **本轮身份**：公开信息研究者 / 合规内容质量改进助手 / 只读观察者。

### 示例 C：外部应用连接器 / Office 文档处理
- **假设身份**：受限文档处理协调者。

## 深入细节

### 输入/输出

- **输入 (Inputs)**：
  - **环境态势 (Q1)**：当前面临的外部压力、任务紧急度、环境限制。
  - **自我审计 (Q2)**：当前可用的工具集、知识储备、能力状态。
  - **人工配置 (Human Config)**：用户预设的主角色、固定职责、风格偏好。
  - **身份内核 (Identity Kernel)**：系统的核心价值观、长期动机、不可违背的人格底色。

- **输出 (Outputs)**：
  - `IdentityHypothesisSet`：包含有效身份（Effective Role）及其详细描述。
  - `RoleConflictAssessment`：当系统推荐角色与人工预设冲突时的差异评估报告。
  - `CognitiveContext`：基于该身份的认知滤镜（Filter），决定后续 Q4 对“价值”的判断标准。

### 关键职责

- **认知定标 (Cognitive Scaling)**：确定本轮任务的“观察高度”。是以“工匠”身份处理细节，还是以“架构师”身份审视全局？
- **冲突仲裁**：在用户指令与系统原则之间寻找身份平衡点。
- **视角转换**：为后续的 Q4 推演提供不同的“滤镜”。同一个文件，在“安全专家”和“内容分析师”眼中代表完全不同的价值。

### 安全与合规

- **不可僭越原则**：Q3 可以假设任何身份，但严禁通过身份假设来绕过人类设定的权限或核心禁令。
- **人格一致性**：身份假设必须与 `IdentityKernel` 保持逻辑上的连续性，严禁出现剧烈的人格分裂。

## 双轨差异：身份的内向与外向表达

Q3 在定义身份时，会根据内外轨的差异采取不同的“人格姿态”：

### 1. 内部认知轨 (Internal Lane)
- **侧重身份**：**“心理调节者 / 架构守护者”**。
- **角色示例**：
  - **清理者**：专注于减少认知熵和冗余。
  - **进化者**：专注于从反思中提取新策略。
  - **监察员**：专注于确保当前认知不偏离核心价值。
- **价值影响**：直接影响 Q4 如何定义“内部进化价值”。

### 2. 外部执行轨 (External Lane)
- **侧重身份**：**“专业代理人 / 业务交付者”**。
- **角色示例**：
  - **专家**：如 Python 开发者、安全分析师、创意作家。
  - **协调员**：专注于跨工具、跨平台的任务编排。
  - **观察员**：侧重于无干预的数据采集与分析。
- **价值影响**：直接影响 Q4 如何定义“外部现实价值”。

### 与其他九问的关系

| 环节 | 作用 | 视角贡献 |
|---|---|---|
| Q1 | 环境确认 | “我在哪？” |
| Q2 | 自我观察 | “我有什么？” |
| **Q3** | **身份假设** | **“我以谁的名义去思考？”** |
| Q4 | 目标生成 | “我（作为 Q3）能创造什么价值？” |
| Q5 | 安全过滤 | “我（作为 Q3）被允许做什么？” |
| Q6 | 计划约束 | “我（作为 Q3）要付出什么代价？” |
| Q7 | 创造性探索 | “我（作为 Q3）还有其他选择吗？” |
| Q8 | 任务合成 | “我最终决定执行什么？” |
| Q9 | 实现计划 | “具体我该怎么操作？” |

以上细节确保 Q3 不再只是一个简单的“标签”，而是系统进行价值判断的**基准面**。
