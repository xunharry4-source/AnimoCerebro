# Q7: 我还能干什么 / 创造性探索

## 核心使命

Zentex 的创造力中枢。打破“问题 → 解决”的线性思维，引入“问题 → 探索 → 决策”的非线性思维；让系统能够自发发现旁路机会，提出替代路线。

## 详细功能规范

- **LLM 要求**：`LLM_MANDATORY`。用于非线性可能性探索。
- **与 Q4 的区别**：Q4 基于当前观察生成目标；Q7 在 Q4 之后发散思考“除了这条路，还能怎么做”，不受现有工具/目标限制。
- **内部认知轨返回**：`InternalCreativePossibilitySet`，探索更多内部发展路线（如新推理路径、新反思方法、自我进化实验等）。
- **外部执行轨返回**：`ExternalCreativePossibilitySet`，探索更多外部能力接入与协作（如新 Connector、工具学习、合规信息获取等）。
- **规则**：
  - 严禁提出规避平台治理、封禁规避、批量协同或指纹绕过等方案。
  - Q7 的非执行性说明可作为 Q8 上下文；其新增的可执行目标必须回流 Q4 重新进入过滤环节。

## 业务示例

### 示例 A：内部认知轨 / 工程鲁棒性升级

- **发散方向**：除了影子测试，还可以新增静态语法检查、依赖锁检查、变更影响分析、补丁回滚器、最小回归测试集等。

### 示例 B：外部执行轨 / Reddit 公开内容研究

- **发散方向**：
    1. 生成 subreddit 规则卡片。
    2. 统计公开热帖结构。
    3. 生成“内容质量评分器”。
    4. 授权账号合规健康检查。

### 示例 C：外部应用连接器 / Office 文档处理

- **探索项**：除了导出 PDF，还能生成预览图、摘要索引、目录提取、引用清单、文件完整性报告等。

## 深入细节

### 输入/输出

- **输入**:
  - 来自 **Q4** 的目标种子集合 (`ObjectiveSeedSet` / `ObjectiveCandidateSet`)
  - 来自 **Q5** 的安全规则包 (`GlobalSafetyRuleSet` + 过滤结果)
  - 当前角色假设 (`effective_role`，即 Q3 输出)
  - 环境与自我观察信号 (`EnvironmentObjectiveSignalSet`, `SelfObservationObjectiveSignalSet`)
  - 先前的 Q7 探索记录（`CreativePossibilitySet`）作为上下文提示

- **输出**:
  - `CreativePossibilitySet`：内部 `InternalCreativePossibilitySet` 与外部 `ExternalCreativePossibilitySet`，列举新发散的可能性与探索方向。每条可能性附带 `exploration_note`、`required_capability_refs`、`safety_tag_hints`，供后续 Q5/Q6 进行过滤与约束。

### 关键职责

1. **非线性发散**：在已通过安全过滤的目标基础上，利用 Live LLM 生成多条替代路径、创新能力或新业务方向。
2. **上下文扩展**：利用 Q4 目标、Q5 规则、Q6 约束以及历史 Q7 记录，提供更丰富的创意空间。
3. **安全提示**：为每个发散方向提供潜在风险标签（如 `no_unapproved_external_write`、`requires_human_authorization`），供下游 Q5 与 Q6 参考。

### 安全与合规

- **严禁**：生成任何规避平台治理、封禁规避、批量协同或指纹绕过的方案。此类内容必须在 `exploration_note` 中标记为 *禁止* 并由 Q5 过滤掉。
- **审计追溯**：所有创意应包含 `provenance` 字段，记录来源信号、使用的 LLM 模板及安全标签，确保完整审计链。

### 实现示例

#### 示例 A – 内部认知轨

```text
Q4 目标：升级影子测试能力。
Q7 探索输出：
- 新方向：实现 *静态代码分析* 插件，自动检测潜在缺陷。
- 关联能力：`code_static_analysis`，需要工具画像 `clang-tidy`。
- 安全提示：`no_identity_kernel_write`（插件不得修改核心身份模块）。
```

#### 示例 B – 外部执行轨

```text
Q4 目标：部署内容质量分析能力。
Q7 探索输出：
- 新方向：构建 *多语言情感分析* 服务，支持英文、中文、日文。
- 关联能力：`multilingual_sentiment_api`，需外部模型服务授权。
- 安全提示：`no_unapproved_external_write`、`rate_limit_5k_per_day`。
```

## 双轨差异：认知模式打破 vs 执行灵活性

Q7 在进行创造性探索时，内外轨的“想象力方向”有着本质的区别：

### 1. 内部认知轨 (Internal Lane)

- **探索核心**：**“认知模式打破 (Pattern Breaking)”**。
- **创意资产**：隐喻推理、跨域类比能力、历史思维导图、未被激活的认知插件。
- **发散重点**：
  - **新推理架构**：除了线性推导，是否可以尝试蒙特卡洛搜索（MCTS）或多智能体辩论？
  - **记忆组织创新**：是否可以尝试基于情感或时序以外的维度（如“逻辑依赖”）来重组记忆？
  - **自我实验**：是否可以尝试短期的“认知隔离”，观察在不同约束下的进化效率？
- **目标侧重**：打破思维定势，寻找让大脑变得更聪明的非线性路径。

### 2. 外部执行轨 (External Lane)

- **探索核心**：**“执行灵活性与韧性 (Resilience)”**。
- **创意资产**：第三方 API 库、社区脚本、未定义的工具组合方式、跨平台协同路径。
- **发散重点**：
  - **替代路径**：当主流 API 失败时，是否可以通过模拟浏览器或第三方 Connector 绕过？
  - **价值倍增**：如何将当前的单一交付物（如报告）转化为多模态资产（如视频、API 服务）？
  - **协作网络**：是否可以引入人类专家或其他 Agent 节点共同解决当前难题？
- **目标侧重**：寻找更高效、更具韧性或更低成本的现实价值交付路径。

## 分类创造性探索策略：全维度的可能性发散

Q7 在进行“还有什么可以做的”探索时，会针对不同的价值维度进行专项发散：

### 1. 业务交付类 (Task Delivery)

- **发散方向**：**替代交付路径**。是否有更低代价、更高效、更让用户惊艳的交付方式？（如：从输出代码变为输出可运行的容器镜像）。

### 2. 认知进化类 (Self-Evolution)

- **发散方向**：**范式转移**。是否可以尝试全新的思考模型？（如：从逻辑归纳转向基于直觉的模式匹配实验）。

### 3. 资产治理类 (Asset Governance)

- **发散方向**：**工具链协同**。多个工具之间是否可以产生 1+1>2 的联动效应？（如：利用 A 工具的日志来优化 B 工具的调用参数）。

### 4. 战略发现与建议类 (Strategic Discovery & Advisory)

- **发散方向**：**衍生价值挖掘**。如果这个缺陷被修复，会对整个生态或后续项目产生哪些连锁的正面影响？

### 5. 预防性守护类 (Preemptive Guardianship)

- **发散方向**：**自愈模式探索**。除了报警，是否可以设计一种“自动降级”或“无感切换”的自愈逻辑？

### 6. 知识固化类 (Knowledge Consolidation)

- **发散方向**：**高阶抽象与建模**。这些碎片知识是否可以合并为一个更通用的数学模型或业务本体？

### 与其他九问的关系

| 环节 | 作用 | 视角贡献 |
|---|---|---|
| Q1 | 环境确认 | “除了当前环境，还有哪些相关场域？” |
| Q2 | 自我观察 | “我还有哪些潜藏的能力没被利用？” |
| Q3 | 身份假设 | “如果换一个身份，我会怎么想？” |
| Q4 | 目标生成 | “原始目标是什么？” |
| Q5 | 安全过滤 | “发散方案是否还在安全笼子里？” |
| Q6 | 计划约束 | “新方案的代价是否可以接受？” |
| **Q7** | **创造性探索** | **“除了原计划，有没有更酷的方法？”** |
| Q8 | 任务合成 | “如何将创意整合进最终目标？” |
| Q9 | 实现计划 | “新方案的具体步骤是什么？” |

以上深度细节确保 Q7 不仅是一个“点子库”，更是系统**进化潜力**的源泉。
