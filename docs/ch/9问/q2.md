# Q2: 观察自身 / 我现在是什么状态

## 核心使命
大脑自我观察，盘点状态、资产、缺口、异常与可用依赖，并在自身状态已经显示缺陷、低效或升级机会时生成自我观察目标线索。Q2 不是身份结论，也不是最终目标生成；它是自我状态观测与资产盘点层。

## 详细功能规范
- **内部认知轨返回**：`InternalSelfObservationProfile`，包含 `identity_kernel_snapshot`、`memory_health`、`reflection_findings`、`capability_gap_signals`、`strategy_patch_state`、`learning_state`、`reasoning_budget`、`cognitive_plugin_state`、`unresolved_internal_problems`、`self_upgrade_backlog`。
- **内部认知轨职责**：观察大脑自身是否缺少反思能力、记忆聚类能力、价值校准能力、提示价值生成能力、策略提炼能力、纯认知插件、测试能力或自我进化支撑能力。ReflectionEngine 必须能把“自己缺少什么”转成 `CapabilityGapSignal` 进入 Q2。
- **外部执行轨返回**：`ExternalSelfObservationProfile` / `AssetInventory`，包含 `agent_inventory_state`、`cli_inventory_state`、`mcp_inventory_state`、`connector_inventory_state`、`model_provider_state`、`external_service_state`、`auth_scope_state`、`health_probe_state`、`verification_state`、`tool_usage_profiles`、`external_capability_gaps`。
- **外部执行轨职责**：观察外部工具、连接器、服务、Agent、CLI、MCP、Provider 是否存在、是否健康、是否已验证、是否授权、是否具备真实调用证据。必须吸收注册阶段生成的 `ToolUsageProfile`。
- **工具画像装配规则**：CLI/MCP 注册若没有完成 ToolDocumentationLearningService 学习，Q2 必须标记 `profile_missing` 或 `profile_degraded`。
- **规则**：
    - Q2 只能观察和报告，不能生成最终任务、不能自改身份、不能执行外部调用。
    - Q2 可以生成自我观察目标线索，但必须由 Q4 结合 Q1/Q2/能力证据正式目标化。

## 业务示例

### 示例 A：内部认知轨 / 工程鲁棒性升级
- **当前能力**：能读取 src/zentex，能生成候选补丁，能运行部分测试命令。
- **当前缺口**：缺少正式写入前的影子测试流程；失败证据表明测试后置导致运行中断。
- **Q2 单源目标线索**：自身缺少“正式写入前验证能力”，可提示后续 Q4 生成影子测试升级目标。

### 示例 B：外部执行轨 / Reddit 公开内容研究
- **当前能力**：能读取公开页面；能调用内容分析器；可能有 Reddit API 客户端画像。
- **当前缺口**：缺少社区规则结构化解析；缺少公开内容质量评分；缺少人工审核面板。

- **观察项**：读取 ConnectorCapability、permission_scope、health_probe、ToolUsageProfile、历史调用证据和验证状态。

## 深入细节

### 输入/输出

- **输入 (Inputs)**：
  - **静态能力表**：系统加载的插件清单、Agent 画像、工具定义（Tool Definition）。
  - **运行快照**：当前加载的上下文长度、活跃的记忆片段、当前会话的权限令牌状态。
  - **能力证据 (Evidence)**：历史成功/失败的记录摘要（Q2 需要从中推断哪些能力是“真实的”）。
  - **反思反馈**：来自 Reflection 引擎的缺口信号、逻辑矛盾警告。

- **输出 (Outputs)**：
  - `SelfObservationProfile`：对自身能力的结构化审计报告。
  - `CapabilityBoundaryEvidence`：支持后续 Q4 推演的能力证据引用 ID（如“Tool-X 已通过自检”）。
  - `SelfObservationObjectiveSignalSet`：由自身缺陷或潜能触发的目标线索（例如：发现某工具已加载但从未初始化，产生“激活该工具”的线索）。

### 关键职责

- **认知审计 (Cognitive Audit)**：不只是罗列工具，而是评估工具的“健康度”与“可达性”。
- **过程监督 (Process Supervision)**：持续观察内部功能与外部工具（Agent, CLI, MCP）的调用轨迹、成功率及资源消耗。
- **能力映射**：将底层的代码接口映射为高层的认知能力描述。
- **缺口识别**：识别“我以为我能做，但实际证据显示我不能做”的认知不一致点。

### 安全与合规

- **禁止自我修改**：Q2 仅允许观察，严禁在观察过程中修改任何内核参数或工具状态。
- **真实性校验**：所有能力声明必须有证据支撑，严禁将“未验证的插件”报告为“完全可用”。

## 双轨差异：观察维度的侧重

Q2 在审视内部与外部时，其“自我审计”的透镜存在显著差异：

### 1. 内部认知轨 (Internal Lane)
- **观察侧重**：**“自洽性与完整性”**。
- **审计重点**：
  - **认知负荷**：当前上下文是否过载？记忆是否需要压缩？
  - **逻辑一致性**：目前的身份（Q3）与我的核心原则（IdentityKernel）是否冲突？
  - **进化潜能**：我的策略补丁库是否已经过时？
- **产出价值**：为 Q4 提供“自我重构”的素材。

### 2. 外部执行轨 (External Lane)
- **观察侧重**：**“触达力与授权深度”**。
- **审计重点**：
  - **工具链完整性**：完成任务所需的 CLI、MCP 或 Connector 是否已就绪？
  - **权限边界**：我是否有权读写目标目录？网络连接是否被墙？
  - **物理资源**：宿主机剩余空间、API 配额是否足以支撑后续动作？
- **产出价值**：为 Q4 提供“现实干预”的基础。

### 与其他九问的关系

| 环节 | 作用 | 关键产物 |
|---|---|---|
| Q1 | 环境确认 | `EnvironmentConfirmation` |
| **Q2** | **自我观察** | **`SelfObservationProfile` / `Evidence`** |
| Q3 | 身份假设 | `IdentityHypothesisSet` |
| Q4 | 目标生成 | `ObjectiveSeedSet` |
| Q5 | 安全过滤 | `GlobalSafetyRuleSet` |
| Q6 | 计划约束 | `PlanConstraintSet` |
| Q7 | 创造性探索 | `CreativePossibilitySet` |
| Q8 | 任务合成 | `FinalObjectiveProfile` |
| Q9 | 实现计划 | `ImplementationPlanDraft` |

以上深度细节确保 Q2 不再是一个简单的“检查清单”，而是系统深度**自省**的起点。
