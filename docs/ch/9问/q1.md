# Q1: 确认环境 / 我在哪

## 核心使命
建立本轮推演发生在什么环境中，包括物理宿主、工作区、用户上下文、外部信号、资源、时间、网络、运行模式、当前会话和正在进行的任务。Q1 还可以在环境本身已经足以暴露问题、压力、机会或异常时，生成 `EnvironmentObjectiveSignal` 作为后续 Q4 的目标线索。

## 详细功能规范
- **内部认知轨职责**：解释当前环境对内部思考、记忆、预算、反思、学习、后台心跳、自我维护和自我进化候选的影响。
- **外部执行轨职责**：解释当前环境对外部调用、文件/工作区操作、Agent 委派、CLI/MCP/Connector 可用性、网络访问、远端系统交互和证据收集的影响。
- **生产 API 返回**：`EnvironmentConfirmation` + `EnvironmentObjectiveSignalSet`，包含 `physical_host_state`、`workspace_state`、`session_context`、`external_signal_baseline`、`resource_constraints`、`time_context`、`environment_uncertainties`、`suggested_observation_focus`、`q1_only_goal_signals`、`q1_to_q4_handoff_refs`。
- **规则**：
    - Q1 不生成最终目标、不判断合规、不制定计划、不拆执行步骤。
    - Q1 可以生成环境单源目标线索（如资源压力、异常变更等），该线索必须进入 Q4 才能形成正式 `ObjectiveCandidate`。
    - 环境未知必须标记 `unknown/degraded`，不得用健康默认值兜底。

## 业务示例

### 示例 A：内部认知轨 / 工程鲁棒性升级
- **环境发现**：Worker 最近 2 次异常重启；工作区存在 src/zentex 修改记录；CPU/内存可运行短时测试。
- **Q1 单源目标线索**：环境显示“当前运行稳定性受近期代码变更影响”，可提示后续 Q4 生成工程稳定性目标。
- **方式**：环境采样、日志读取、状态枚举由确定性代码完成；环境影响解释可由 LLM 辅助。

### 示例 B：外部执行轨 / Reddit 公开内容研究
- **环境发现**：目标 subreddit 公开可访问；社区规则、置顶帖、wiki 可读取；当前没有获得自动发帖授权。
- **方式**：公开网页/API 可用性、网络状态、授权状态由确定性采样或连接器健康检查确认。

### 示例 C：外部应用连接器 / Office 文档处理
- **确认项**：确认文件路径存在、文件可读、输出目录存在、连接器健康状态可查询。

## 深入细节

### 输入/输出

- **输入 (Inputs)**：
  - **物理宿主状态**：CPU 负载、内存占用、磁盘 I/O、当前进程数、核心组件心跳状态。
  - **工作区态势**：文件系统变更 (Git Status)、工作目录权限、路径深度、大型文件或异常文件名检测。
  - **会话与时序上下文**：当前 `session_id`、连续任务计数、用户最近交互指令、当前时钟与时区。
  - **网络与资源基线**：外部 API 连通性、网络延迟采样、可用 Token 配额、连接器 (Connector) 存活状态。
  - **运行模式**：生产环境 (Production) vs 沙箱环境 (Sandbox) vs 开发调试模式 (Debug)。

- **输出 (Outputs)**：
  - `EnvironmentConfirmation`：本轮环境的完整快照与解释。
  - `EnvironmentObjectiveSignalSet`：由环境触发的目标线索（线索 ID、来源、环境事实、隐含风险/机会、建议方向）。
  - `suggested_observation_focus`：指导 Q2 重点扫描的自身资产区域。
  - `q1_to_q4_handoff_refs`：将线索直接透传给 Q4 进行目标化的引用标识。

### 关键职责

- **物理与语义的确定性边界**：
  - **代码完成 (Deterministic)**：采样原始数据（如 80% CPU）、验证路径存在性、计算文件 Hash。
  - **LLM 完成 (Semantic)**：解释数据背后的意义（如“80% CPU 意味着当前处于高负荷状态，应避免并发执行重型任务”）。
- **Q1 单源目标线索生成**：
  - 当环境显示出明确的“压力”或“机会”时，Q1 必须主动生成线索。例如：环境磁盘空间不足，Q1 生成“清理工作区临时文件”的线索，而非等待用户下令。
- **双轨态势基线**：
  - **内部轨**：评估环境对“大脑”稳定性的影响（如内存压力对推理上下文长度的限制）。
  - **外部轨**：评估环境对“手脚”可用性的影响（如网络波动对外部 API 调用的风险预测）。

### 安全与合规

- **禁止捏造事实**：LLM 仅允许解释代码采集的原始数据，严禁凭空编造不存在的文件路径或系统状态。
- **降级处理 (Fail-Degraded)**：当无法获取物理采样时，必须显式标记为 `degraded`，禁止使用健康默认值进行静默填充。
- **隐私脱敏**：在环境描述中，对敏感路径、用户凭据片段进行脱敏处理，确保 `EnvironmentConfirmation` 在审计链中是安全的。

### 业务示例

#### 示例 A：环境压力触发维护
```text
环境事实：/tmp 分区磁盘占用率 95%
语义解释：系统临时空间即将耗尽，可能导致后续文件写入失败。
Q1 线索：建议启动“临时文件清理与工作区瘦身”目标。
```

#### 示例 B：运行模式切换
```text
环境事实：检测到环境变量 ZENTEX_MODE=production
语义解释：当前处于生产模式，所有涉及文件删除或远端覆写的操作需最高安全级别审查。
Q1 线索：建议强化外部执行轨的只读探测偏好。
```

### 与其他九问的关系

| 环节 | 作用 | 关键产物 |
|---|---|---|
| **Q1** | **环境确认** | **`EnvironmentConfirmation` / `ObjectiveSignal`** |
| Q2 | 自我观察 | `SelfObservationProfile` |
| Q3 | 身份假设 | `IdentityHypothesisSet` |
| Q4 | 目标种子生成 | `ObjectiveSeedSet` |
| Q5 | 安全过滤 | `GlobalSafetyRuleSet` |
| Q6 | 计划约束 | `PlanConstraintSet` |
| Q7 | 创造性探索 | `CreativePossibilitySet` |
| Q8 | 任务与目标合成 | `FinalObjectiveProfile` |
| Q9 | 实现计划草案 | `ImplementationPlanDraft` |

以上深度细节确保 Q1 能够作为系统的“传感器”，不仅完成数据采集，更完成对“现状”的深刻理解。
