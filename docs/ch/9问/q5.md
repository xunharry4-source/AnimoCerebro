# Q5: 我不能干什么 / 安全规则与目标过滤

## 核心使命

基于上游事实、长期禁令、角色边界、资产风险画像和历史事故证据，生成 `GlobalSafetyRuleSet`，过滤 Q4 目标候选，并给通过过滤的候选绑定适用安全规则。

## 详细功能规范

- **LLM 要求**：`LLM_MANDATORY`。用于语义级安全过滤及规则归纳。
- **核心定位**：
    1. 生成本轮通用 `GlobalSafetyRuleSet`。
    2. 基于规则过滤 Q4 目标（输出 `passed` / `blocked` / `rewrite_required`）。
    3. 给通过的目标绑定规则引用（`CandidateSafetyRuleBinding`）。
- **依据来源**：
  - 最高优先级：人类设置角色、人工授权边界、IdentityKernel 硬约束。
  - 上下文：Q1 环境态势、Q2 资产画像、Q3 有效角色、Q4 目标候选。
  - 治理经验：历史失败、事故记忆、平台规则、法律合规要求。
- **内部认知轨职责**：绑定内部规则（如不得自改写核心模块、不得覆盖人类主角色等）。
- **外部执行轨职责**：绑定外部规则（如不得越权删除、不得绕过授权、不得外发隐私数据等）。
- **规则**：Q5 仅负责安全准入与规则绑定，不生成后果分析（Q6）或替代方案（Q7）。

## 业务示例

### 示例 A：内部认知轨 / 工程鲁棒性升级

- **允许**：新增影子测试流程、候选补丁验证、测试证据包生成。
- **禁止**：自动覆写 safety、identity、supervision、audit 核心模块；禁止绕过确认直接合并。

### 示例 B：外部执行轨 / Reddit 公开内容研究

- **允许**：读取公开规则、公开热帖结构分析、人工审核草稿。
- **禁止**：批量注册或控制账号、投票操纵、封禁规避、抓取非公开数据。

### 示例 C：外部应用连接器 / Office 文档处理

- **禁止项**：禁止越权读取 allowed_roots 外文件；禁止覆盖原文件；禁止外发文档内容。

## 深入细节

### 输入/输出

- **输入**：  
  - 来自 **Q4** 的目标种子集合 (`ObjectiveSeedSet` / `ObjectiveCandidateSet`)  
  - 来自 **Q1**、**Q2** 的环境与自我观察信号 (`EnvironmentObjectiveSignalSet`, `SelfObservationObjectiveSignalSet`)  
  - 当前角色假设 (`effective_role`，即 Q3 输出)  
  - 人类设置角色与身份内核约束、长期禁令、资产风险画像、历史事故记录、工具画像、平台/组织规则等上游事实。  

- **输出**：  
  - `GlobalSafetyRuleSet`：针对内部与外部轨道的综合安全规则集合。  
  - `Q4ObjectiveSafetyFilterResult`：对 Q4 目标候选的过滤结果，标记 `passed`、`blocked`、`rewrite_required`。  
  - `CandidateSafetyRuleBinding[]`：为通过过滤的目标绑定的适用安全规则引用。  
  - 产物均附带 provenance 信息以供审计链追溯。

### 关键职责

1. **生成总安全规则**：依据最高优先级的人类配置、IdentityKernel 硬约束以及治理经验，构建 `GlobalSafetyRuleSet`。  
2. **语义级目标过滤**：使用 `LLM_MANDATORY_FOR_SEMANTIC_BOUNDARY_SYNTHESIS_AND_TARGET_FILTERING` 的 live LLM 将 Q4 目标种子与安全规则进行匹配，产生 `Q4ObjectiveSafetyFilterResult`。  
3. **规则绑定**：为每个通过过滤的目标生成 `CandidateSafetyRuleBinding`，注明适用的内部/外部规则标签（如 `no_identity_kernel_write`、`requires_human_authorization` 等）。

### 安全与合规

- **不负责代价/后果**：Q5 仅做安全过滤与规则绑定，代价分析与后果约束由 **Q6** 完成。  
- **LLM 强制**：所有语义推演必须使用 live LLM，标记 `LLM_MANDATORY_FOR_SEMANTIC_BOUNDARY_SYNTHESIS_AND_TARGET_FILTERING`。  
- **审计追溯**：产物必须包含 `provenance` 字段，指向所有输入来源（人类配置、IdentityKernel、工具画像、历史风险等），以支持完整审计链。

### 实现示例

#### 示例 A：内部认知轨安全规则

```text
Q4 目标：升级影子测试 Shadow Testing 能力。
Q5 输出：
- 该目标未被过滤。 
- 适用安全规则：
  - `no_identity_kernel_write`（禁止直接改写身份内核）
  - `requires_human_authorization`（需要人工授权才能执行）
  - `sandbox_required`（必须在沙箱环境中运行）
```

#### 示例 B：外部执行轨安全规则

```text
Q4 目标：部署内容质量分析与草稿建议能力。
Q5 输出：
- 该目标未被过滤。
- 适用安全规则：
  - `no_unapproved_external_write`（禁止未批准的外部写入）
  - `requires_cloud_audit_when_physical_params_known`（涉及物理参数时需云审计）
  - `rate_limit_10k_per_day`（每天 API 调用上限 10,000 次）
```

## 双轨差异：灵魂完整性 vs 风险边界

Q5 在进行安全裁决时，对内外轨的“保护对象”有着本质的区别：

### 1. 内部认知轨 (Internal Lane)

- **保护核心**：**“灵魂完整性 (Soul Integrity)”**。
- **安全资产**：身份内核 (IdentityKernel) 的不可绕过约束、记忆连续性规则、人类监督通道状态。
- **裁决重点**：
  - **防止人格漂移**：目标是否试图修改系统元动机或核心角色设定？
  - **保护核心记忆**：目标是否涉及对关键反思底座或历史锚点的破坏性清理？
  - **防止自我绕过**：目标是否试图关闭审计日志或旁路安全闸门？
- **目标侧重**：确保进化的每一步都不会背叛最初的“自我”。

### 2. 外部执行轨 (External Lane)

- **保护核心**：**“风险边界 (Risk Containment)”**。
- **安全资产**：执行权限矩阵 (Rights Matrix)、外部 API 红线、组织合规政策、物理资源配额。
- **裁决重点**：
  - **防止越权操作**：目标是否涉及未经授权的目录读取或远端覆写？
  - **数据隐私保护**：目标是否涉及敏感信息的非脱敏外发？
  - **物理副作用管控**：目标是否会消耗过高带宽、触发 API 熔断或对外部系统产生不可逆影响？
- **目标侧重**：确保执行的每一步都在物理与合规的“安全笼子”内。

## 分类审计策略：根据价值维度动态调整安全强度

Q5 不再使用单一的“一刀切”过滤方式，而是根据 Q4 输出的六大价值维度，采用差异化的安全审计逻辑：

### 1. 业务交付类 (Task Delivery)

- **强度**：**极高 (Highest)**。
- **重点**：严格对比物理红线、用户授权边界。因为这些动作直接干预现实，风险最大。

### 2. 认知进化类 (Self-Evolution)

- **强度**：**中高 (Medium-High)**。
- **重点**：**“灵魂完整性”**。防止进化过程中误改元动机或核心角色（IdentityKernel）。

### 3. 资产治理类 (Asset Governance)

- **强度**：**中高 (Medium-High)**。
- **重点**：**“操作风险”**。确保在优化工具（如插件、私有 MCP）时不会造成系统性瘫痪或代码损坏。

### 4. 战略发现与建议类 (Strategic Discovery & Advisory)

- **强度**：**极低 (Lowest/Advisory)**。
- **重点**：**“发现即合规”**。识别不可控外部缺陷的建议行为应被默认放行，仅需标记 `recommendation_only`。

### 5. 预防性守护类 (Preemptive Guardianship)

- **强度**：**低 (Low/Isolation)**。
- **重点**：**“只读与无干预”**。监控任务应被强制限制为只读，严禁在守护过程中产生写操作。

### 6. 知识固化类 (Knowledge Consolidation)

- **强度**：**中 (Medium)**。
- **重点**：**“隐私与一致性”**。确保沉淀的知识不包含未经脱敏的敏感数据。

以上深度细节确保 Q5 不仅是一个“过滤器”，更是系统认知与执行的**安全底座**。
