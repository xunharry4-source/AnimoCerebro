# Q8: What Do I Finally Arrange / Formulate Tasks & Goals

## Core Mission
Task and goal formation hub. Integrate Q4 goal candidates, Q5 safety rule packages, Q6 constraint/consequence conditions, and Q7 additional exploration to form the final tasks and goal combinations to be advanced this round.

## Detailed Functional Specifications
- **LLM Requirement**: `LLM_MANDATORY`. Comprehensive multi-party input for task decision-making.
- **Input Dependencies**: Must reference Q4 (CandidateSet), Q5 (SafetyContract), Q6 (ConstraintSet), Q7 (PossibilitySet). Task design will be blocked if Q5/Q6 are missing.
- **Internal Cognitive Lane Returns**: `InternalTaskDesignProfile`, containing selected/rejected/delayed internal goals, as well as cognitive, memory, learning, and other task groups.
- **External Execution Lane Returns**: `ExternalTaskDesignProfile`, containing selected/rejected/delayed external goals, as well as agent delegation, CLI/MCP invocation, approval, evidence collection, and other task groups.
- **Decision Rules**:
    - Q8 can merge, split, sort, or delay Q4 goals.
    - Must consume Q5 rule packages, rejecting or temporarily buffering conflicting goals.
    - Must transform Q6 constraints (precautions, rollback requirements, etc.) into part of task design.
- **Rules**: Q8 does not formulate underlying implementation steps, does not generate physical task_id or resource locks.

## Business Examples

### Example A: Internal Cognitive Lane / Engineering Robustness Upgrade
- **Final Task Goals**: Prioritize upgrading shadow testing capabilities; simultaneously add static syntax checking and EvidenceBundle generation as supporting goals; defer rollbacker implementation.
- **Constraint Inheritance**: Inherit Q5 safety clauses and Q6 sandbox, verification requirements.

### Example B: External Execution Lane / Reddit Public Content Research
- **Final Task Goals**: First perform public rule learning and content quality scorer; generate manual review templates; do not implement automatic posting or evasion functions.
- **Constraint Inheritance**: Inherit Q5 compliance prohibitions and Q6 read-only, manual review requirements.

### Example C: External Application Connector / Office Document Processing
- **Decision Conclusion**: Choose "read docx + generate summary + export PDF + generate evidence package," defer preview image generation.

## In-Depth Details

### Inputs/Outputs

- **Inputs**:
  - Objective seed sets from **Q4** (`ObjectiveSeedSet` / `ObjectiveCandidateSet`)
  - Safety rule packages from **Q5** (`GlobalSafetyRuleSet` + filtering results)
  - Plan constraints from **Q6** (`PlanConstraintSet`)
  - Creative exploration sets from **Q7** (`CreativePossibilitySet`)
  - Current role hypothesis (`effective_role`, i.e., Q3 output)
  - Environmental and self-observation signals (`EnvironmentObjectiveSignalSet`, `SelfObservationObjectiveSignalSet`)

- **Outputs**:
  - `FinalObjectiveProfile`: Comprehensive internal `InternalTaskDesignProfile` and external `ExternalTaskDesignProfile`, defining final task and goal combinations, including inherited safety rules, constraint conditions, and executable sub-goal lists.
  - Each task entry includes `provenance` information for audit traceability when Q9 generates implementation plans.

### Key Responsibilities

1. **Task Aggregation**: Under the constraints of safety filtering (Q5) and plan constraints (Q6), aggregate goal seeds generated by Q4 and combine with Q7's innovative directions to form complete task sets.
2. **Constraint Inheritance**: Automatically embed Q5 safety tags, Q6 consequence constraints, and Q7 exploration hints into task descriptions to ensure downstream implementation plans comply with all rules.
3. **Priority & Scheduling**: Sort, merge, or delay tasks based on resource availability, risk assessment, and business value.

### Security & Compliance

- **Rule Compliance**: All tasks must comply with prohibitions in `GlobalSafetyRuleSet`; violating goals will be automatically removed or marked as `delayed`.
- **Audit Traceability**: `FinalObjectiveProfile` must record source signals (Q4, Q5, Q6, Q7) and safety tags for each task, forming a complete chain for Q9 use.

## Dual-Lane Differences: Cognitive Blueprint vs. Action Blueprint

Q8 has essential attribute differences in its "final state products" when synthesizing goals for internal vs. external lanes:

### 1. Internal Cognitive Lane
- **Synthesis Core**: **"Cognitive Blueprint"**.
- **Synthesis Assets**: Q4 self-evolution goals, Q5 soul protection rules, Q6 logical stability constraints.
- **Synthesis Focus**:
  - **Causal Consistency**: Ensure synthesized task groups do not cause logical deadlocks within the brain.
  - **Evolution Path Alignment**: Ensure this round's tasks pave the way for future cognitive upgrades, not just temporary patches.
  - **Reflection Closure**: Ensure each task includes a "post-execution feedback" mechanism to flow back to the Reflection engine.
- **Goal Emphasis**: Build an internal engineering set that enhances the system's "mental maturity."

### 2. External Execution Lane
- **Synthesis Core**: **"Action Blueprint"**.
- **Synthesis Assets**: Q4 business value goals, Q5 physical compliance prohibitions, Q6 ROI constraints, Q7 flexibility suggestions.
- **Synthesis Focus**:
  - **Deliverable Clarity**: Each sub-goal must correspond to a clearly observable artifact.
  - **Execution Chain Optimization**: Seek shortest-path task orchestration while satisfying safety and cost requirements.
  - **Evidence Integrity**: Ensure each external action is bound to necessary audit evidence collection tasks.
- **Goal Emphasis**: Build an action package that produces actual physical influence and solves real-world problems.
