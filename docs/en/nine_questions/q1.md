# Q1: Confirm Environment / Where Am I

## Core Mission
Establish what environment this round of reasoning takes place in, including physical host, workspace, user context, external signals, resources, time, network, runtime mode, current session, and ongoing tasks. Q1 can also generate `EnvironmentObjectiveSignal` as goal clues for subsequent Q4 when the environment itself already exposes problems, pressure, opportunities, or anomalies.

## Detailed Functional Specifications
- **Internal Cognitive Lane Responsibilities**: Explain how the current environment affects internal thinking, memory, budget, reflection, learning, background heartbeat, self-maintenance, and self-evolution candidates.
- **External Execution Lane Responsibilities**: Explain how the current environment affects external calls, file/workspace operations, agent delegation, CLI/MCP/Connector availability, network access, remote system interaction, and evidence collection.
- **Production API Returns**: `EnvironmentConfirmation` + `EnvironmentObjectiveSignalSet`, containing `physical_host_state`, `workspace_state`, `session_context`, `external_signal_baseline`, `resource_constraints`, `time_context`, `environment_uncertainties`, `suggested_observation_focus`, `q1_only_goal_signals`, `q1_to_q4_handoff_refs`.
- **Rules**:
    - Q1 does not generate final goals, judge compliance, formulate plans, or break down execution steps.
    - Q1 can generate environmental single-source goal clues (such as resource pressure, abnormal changes), which must enter Q4 to form formal `ObjectiveCandidate`.
    - Unknown environments must be marked as `unknown/degraded`; healthy default values must not be used as fallback.

## Business Examples

### Example A: Internal Cognitive Lane / Engineering Robustness Upgrade
- **Environmental Discovery**: Worker has restarted abnormally 2 times recently; workspace has modification records in src/zentex; CPU/memory can run short-term tests.
- **Q1 Single-Source Goal Clue**: Environment shows "current operational stability is affected by recent code changes", which can prompt Q4 to generate engineering stability goals.
- **Method**: Environmental sampling, log reading, and state enumeration are completed by deterministic code; environmental impact interpretation can be assisted by LLM.

### Example B: External Execution Lane / Reddit Public Content Research
- **Environmental Discovery**: Target subreddit is publicly accessible; community rules, pinned posts, wiki are readable; currently no authorization for automatic posting.
- **Method**: Public webpage/API availability, network status, and authorization status are confirmed by deterministic sampling or connector health checks.

### Example C: External Application Connector / Office Document Processing
- **Confirmation Items**: Confirm file path exists, file is readable, output directory exists, connector health status is queryable.

## In-Depth Details

### Inputs/Outputs

- **Inputs**:
  - **Physical Host State**: CPU load, memory usage, disk I/O, current process count, core component heartbeat status.
  - **Workspace Posture**: Filesystem changes (Git Status), working directory permissions, path depth, large file or abnormal filename detection.
  - **Session & Temporal Context**: Current `session_id`, consecutive task count, user's most recent interaction commands, current clock and timezone.
  - **Network & Resource Baseline**: External API connectivity, network latency sampling, available token quota, connector survival status.
  - **Runtime Mode**: Production vs Sandbox vs Debug mode.

- **Outputs**:
  - `EnvironmentConfirmation`: Complete snapshot and interpretation of this round's environment.
  - `EnvironmentObjectiveSignalSet`: Goal clues triggered by the environment (clue ID, source, environmental facts, implied risks/opportunities, suggested directions).
  - `suggested_observation_focus`: Guide Q2 to focus scanning on specific self-asset areas.
  - `q1_to_q4_handoff_refs`: Reference identifiers that directly pass clues to Q4 for goal formulation.

### Key Responsibilities

- **Deterministic Boundary Between Physical and Semantic**:
  - **Code Completion (Deterministic)**: Sample raw data (e.g., 80% CPU), verify path existence, calculate file hash.
  - **LLM Completion (Semantic)**: Interpret the meaning behind the data (e.g., "80% CPU means the system is under high load and should avoid concurrent execution of heavy tasks").
- **Q1 Single-Source Goal Clue Generation**:
  - When the environment shows clear "pressure" or "opportunity", Q1 must proactively generate clues. For example: insufficient disk space, Q1 generates a clue for "cleaning up workspace temporary files" rather than waiting for user command.
- **Dual-Lane Posture Baseline**:
  - **Internal Lane**: Assess the environment's impact on "brain" stability (e.g., memory pressure limiting reasoning context length).
  - **External Lane**: Assess the environment's impact on "hands and feet" availability (e.g., network fluctuation risk prediction for external API calls).

### Security & Compliance

- **Prohibit Fabricating Facts**: LLM is only allowed to interpret raw data collected by code; it is strictly prohibited to fabricate non-existent file paths or system states out of thin air.
- **Degraded Handling (Fail-Degraded)**: When physical sampling cannot be obtained, it must be explicitly marked as `degraded`; silent filling with healthy default values is prohibited.
- **Privacy Desensitization**: Sensitive paths and user credential fragments in environmental descriptions must be desensitized to ensure `EnvironmentConfirmation` is safe in the audit chain.

### Business Examples

#### Example A: Environmental Pressure Triggers Maintenance
```text
Environmental Fact: /tmp partition disk usage rate 95%
Semantic Interpretation: System temporary space is about to be exhausted, which may cause subsequent file write failures.
Q1 Clue: Suggest initiating "temporary file cleanup and workspace slimming" goal.
```

#### Example B: Runtime Mode Switching
```text
Environmental Fact: Detected environment variable ZENTEX_MODE=production
Semantic Interpretation: Currently in production mode; all operations involving file deletion or remote overwriting require highest security level review.
Q1 Clue: Suggest strengthening read-only detection preference in external execution lane.
```
