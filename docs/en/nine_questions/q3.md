# Q3: Hypothesize Identity / What Identity Should I Use to Understand This Round's Task

## Core Mission
Based on Q1 environment and Q2 self-observation, generate interpretable identity/role hypotheses for this round to provide perspective for Q4 goal generation. If conflicting with human-set roles, the human-set role is the final effective role; Q3 must display differences but must not override.

## Detailed Functional Specifications
- **Input Contract**: Q3 must consume complete structured analysis results from Q1 and Q2. It is prohibited to consume deduction results from Q4 and later modules (prohibit temporal pollution).
- **Highest Priority Rule: Human-Set Role Takes Precedence**: If `human_configured_role` already exists in manual configuration, that role is always the authoritative source of this round's `effective_role`.
- **Differences Must Be Visible**: When AI-inferred role differs from human settings, it must output `RoleConflictAssessment` / `RoleDeltaReport`, clearly displaying the difference.
- **Internal Cognitive Lane Returns**: `InternalIdentityHypothesisSet`, containing valid internal roles, AI-inferred candidates, reasoning, identity kernel references, and difference reports.
- **External Execution Lane Returns**: `ExternalExecutionIdentityHypothesisSet`, containing valid external postures, delegation postures, representation authority limits, and difference reports.
- **Conflict Handling**:
    - No manual setting: Q3 recommended role serves only as a candidate.
    - Substantial conflict exists: Must mark `requires_human_review=true`, but still defer to human settings.
- **Rules**: Q3 does not rewrite IdentityKernel, does not change main role, does not generate goals.

## Business Examples

### Example A: Internal Cognitive Lane / Engineering Robustness Upgrade
- **This Round's Identity**: Internal engineering maintainer / free evolution candidate planner.

### Example B: External Execution Lane / Reddit Public Content Research
- **This Round's Identity**: Public information researcher / compliant content quality improvement assistant / read-only observer.

### Example C: External Application Connector / Office Document Processing
- **Hypothesized Identity**: Restricted document processing coordinator.

## In-Depth Details

### Inputs/Outputs

- **Inputs**:
  - **Environmental Posture (Q1)**: Current external pressures, task urgency, environmental constraints.
  - **Self-Audit (Q2)**: Currently available toolset, knowledge reserves, capability status.
  - **Manual Configuration (Human Config)**: User-preset main role, fixed responsibilities, style preferences.
  - **Identity Kernel**: System's core values, long-term motivations, non-violable personality foundation.

- **Outputs**:
  - `IdentityHypothesisSet`: Contains effective role and its detailed description.
  - `RoleConflictAssessment`: Difference assessment report when system-recommended role conflicts with manual presets.
  - `CognitiveContext`: Cognitive filter based on this identity, determining subsequent Q4's judgment criteria for "value."

### Key Responsibilities

- **Cognitive Scaling**: Determine the "observation height" for this round's task. Process details as a "craftsman" or examine the big picture as an "architect"?
- **Conflict Arbitration**: Find identity balance points between user instructions and system principles.
- **Perspective Shifting**: Provide different "filters" for subsequent Q4 deduction. The same file represents completely different value in the eyes of a "security expert" vs. a "content analyst."

### Security & Compliance

- **Non-Transgression Principle**: Q3 can hypothesize any identity, but it is strictly prohibited to bypass human-set permissions or core prohibitions through identity hypothesis.
- **Personality Consistency**: Identity hypotheses must maintain logical continuity with `IdentityKernel`; drastic personality splits are strictly prohibited.

## Dual-Lane Differences: Introverted vs. Extroverted Expression of Identity

Q3 adopts different "personality postures" when defining identity based on internal vs. external lane differences:

### 1. Internal Cognitive Lane
- **Focus Identity**: **"Psychological Regulator / Architecture Guardian"**.
- **Role Examples**:
  - **Cleaner**: Focuses on reducing cognitive entropy and redundancy.
  - **Evolver**: Focuses on extracting new strategies from reflection.
  - **Inspector**: Focuses on ensuring current cognition does not deviate from core values.
- **Value Impact**: Directly affects how Q4 defines "internal evolution value."

### 2. External Execution Lane
- **Focus Identity**: **"Professional Agent / Business Deliverer"**.
- **Role Examples**:
  - **Expert**: Such as Python developer, security analyst, creative writer.
  - **Coordinator**: Focuses on cross-tool, cross-platform task orchestration.
  - **Observer**: Emphasizes non-intervention data collection and analysis.
- **Value Impact**: Directly affects how Q4 defines "external reality value."
