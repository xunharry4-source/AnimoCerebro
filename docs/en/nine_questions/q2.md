# Q2: Observe Self / What State Am I In Now

## Core Mission
Brain self-observation, inventorying state, assets, gaps, anomalies, and available dependencies, and generating self-observation goal clues when the system's own state already shows defects, inefficiencies, or upgrade opportunities. Q2 is not an identity conclusion nor final goal generation; it is the self-state observation and asset inventory layer.

## Detailed Functional Specifications
- **Internal Cognitive Lane Returns**: `InternalSelfObservationProfile`, containing `identity_kernel_snapshot`, `memory_health`, `reflection_findings`, `capability_gap_signals`, `strategy_patch_state`, `learning_state`, `reasoning_budget`, `cognitive_plugin_state`, `unresolved_internal_problems`, `self_upgrade_backlog`.
- **Internal Cognitive Lane Responsibilities**: Observe whether the brain itself lacks reflection capability, memory clustering capability, value calibration capability, prompt value generation capability, strategy extraction capability, pure cognitive plugins, testing capability, or self-evolution support capability. ReflectionEngine must be able to convert "what am I missing" into `CapabilityGapSignal` entering Q2.
- **External Execution Lane Returns**: `ExternalSelfObservationProfile` / `AssetInventory`, containing `agent_inventory_state`, `cli_inventory_state`, `mcp_inventory_state`, `connector_inventory_state`, `model_provider_state`, `external_service_state`, `auth_scope_state`, `health_probe_state`, `verification_state`, `tool_usage_profiles`, `external_capability_gaps`.
- **External Execution Lane Responsibilities**: Observe whether external tools, connectors, services, agents, CLI, MCP, providers exist, are healthy, verified, authorized, and have real invocation evidence. Must absorb `ToolUsageProfile` generated during registration phase.
- **Tool Profile Assembly Rules**: If CLI/MCP registration has not completed ToolDocumentationLearningService learning, Q2 must mark `profile_missing` or `profile_degraded`.
- **Rules**:
    - Q2 can only observe and report; it cannot generate final tasks, self-modify identity, or execute external calls.
    - Q2 can generate self-observation goal clues, but they must be formally goalized by Q4 in combination with Q1/Q2/capability evidence.

## Business Examples

### Example A: Internal Cognitive Lane / Engineering Robustness Upgrade
- **Current Capability**: Can read src/zentex, can generate candidate patches, can run some test commands.
- **Current Gap**: Lacks formal pre-write shadow testing process; failure evidence shows post-testing leads to runtime interruption.
- **Q2 Single-Source Goal Clue**: System lacks "pre-formal-write verification capability", which can prompt Q4 to generate shadow testing upgrade goals.

### Example B: External Execution Lane / Reddit Public Content Research
- **Current Capability**: Can read public pages; can invoke content analyzers; may have Reddit API client profile.
- **Current Gap**: Lacks structured parsing of community rules; lacks public content quality scoring; lacks manual review panel.
- **Observation Items**: Read ConnectorCapability, permission_scope, health_probe, ToolUsageProfile, historical invocation evidence, and verification status.

## In-Depth Details

### Inputs/Outputs

- **Inputs**:
  - **Static Capability Table**: System-loaded plugin inventory, agent profiles, tool definitions (Tool Definition).
  - **Runtime Snapshot**: Currently loaded context length, active memory fragments, current session's permission token status.
  - **Capability Evidence**: Summary of historical success/failure records (Q2 needs to infer which capabilities are "real" from this).
  - **Reflection Feedback**: Gap signals from Reflection engine, logical contradiction warnings.

- **Outputs**:
  - `SelfObservationProfile`: Structured audit report of self-capabilities.
  - `CapabilityBoundaryEvidence`: Capability evidence reference IDs supporting subsequent Q4 deduction (e.g., "Tool-X has passed self-check").
  - `SelfObservationObjectiveSignalSet`: Goal clues triggered by self-defects or potential (e.g., discovering a tool is loaded but never initialized, producing a clue to "activate that tool").

### Key Responsibilities

- **Cognitive Audit**: Not just listing tools, but evaluating the "health" and "accessibility" of tools.
- **Process Supervision**: Continuously observe invocation traces, success rates, and resource consumption of internal functions and external tools (Agent, CLI, MCP).
- **Capability Mapping**: Map low-level code interfaces to high-level cognitive capability descriptions.
- **Gap Identification**: Identify cognitive inconsistencies where "I thought I could do it, but actual evidence shows I cannot."

### Security & Compliance

- **Prohibit Self-Modification**: Q2 is only allowed to observe; it is strictly prohibited to modify any kernel parameters or tool states during observation.
- **Authenticity Verification**: All capability claims must be supported by evidence; it is strictly prohibited to report "unverified plugins" as "fully available."

## Dual-Lane Differences: Focus Dimensions of Observation

Q2 has significant differences in its "self-audit" lens when examining internal vs. external aspects:

### 1. Internal Cognitive Lane
- **Observation Focus**: **"Consistency and Completeness"**.
- **Audit Priorities**:
  - **Cognitive Load**: Is the current context overloaded? Does memory need compression?
  - **Logical Consistency**: Does the current identity (Q3) conflict with my core principles (IdentityKernel)?
  - **Evolution Potential**: Has my strategy patch library become outdated?
- **Output Value**: Provides material for Q4's "self-restructuring."

### 2. External Execution Lane
- **Observation Focus**: **"Reachability and Authorization Depth"**.
- **Audit Priorities**:
  - **Toolchain Completeness**: Are the CLI, MCP, or Connectors required to complete the task ready?
  - **Permission Boundaries**: Do I have rights to read/write target directories? Is the network connection blocked?
  - **Physical Resources**: Is the host's remaining space and API quota sufficient to support subsequent actions?
- **Output Value**: Provides foundation for Q4's "reality intervention."
