# AnimoCerebro 项目总结（基于当前代码）

本文档基于当前仓库实际代码、页面入口、后端路由与测试分布整理，重点说明：

- 当前已经落地的功能
- 当前主要目录与职责
- 哪些模块已经具备实现，哪些仍偏骨架/预留

更新时间：2026-04-06

## 1. 项目当前定位

当前项目并不是单一前端或单一后端，而是一个由以下部分组成的综合系统：

- `zentex runtime`：负责运行时、会话、九问认知链、工具编排、Transcript 记录等核心能力
- `web_console`：基于 FastAPI 的后端控制台接口层，向前端暴露运行态、任务、插件、审计、九问等能力
- `admin-portal`：基于 React + Vite 的管理前端，提供实时指挥台、九问页面、任务/Agent/插件/MCP 管理等 UI
- `plugins`：插件与外部能力层，承载九问插件、感知插件、执行插件、模型 Provider、记忆/模拟/权重等扩展能力

从当前代码看，项目已经进入“可运行骨架 + 多模块联调 + 管理台可视化 + 测试覆盖较完整”的阶段，而不是纯概念设计阶段。

## 2. 当前已实现的核心功能

### 2.1 运行时主链

`src/zentex/runtime` 已实现项目最核心的一批运行时能力：

- `BrainRuntime`
  - 负责装配运行时依赖
  - 管理会话实例
  - 维护运行时状态快照
  - 维护九问状态与冷启动初始化状态

- `BrainSession`
  - 负责单个会话的连续性状态管理
  - 支持从 transcript 恢复上下文
  - 持有最近一轮工作记忆、元认知、模拟结果、决策结果等

- `ThinkLoop`
  - 已实现固定九阶段认知执行骨架
  - 包含观察、框架化、工作状态刷新、风险检测、模拟、元认知、工具编排、决策综合、巩固九个阶段
  - 已具备冷启动九问推理入口
  - 在关键阶段接入真实 LLM Provider 调用

- `BrainTranscriptStore`
  - 使用 JSONL append-only 方式保存运行事件
  - 支持按 `session_id` / `turn_id` 读取
  - 为回放、恢复、审计提供事实来源

### 2.2 九问认知链

九问主线不是只存在于文档中，当前代码已经形成实际模块与插件体系：

- 运行时存在 `nine_questions` 路由、状态、执行器、启动快照
- 插件层存在 Q1-Q9 对应插件目录
- 前端存在九问总览页、详情页、测试页、沙箱页
- Web Console 存在九问接口路由

结合当前代码，可以确认已实现的能力包括：

- 九问状态缓存与刷新
- 冷启动触发九问推理
- 九问问题插件化挂载
- 九问页面化展示
- 九问生产证据/测试页拆分展示

### 2.3 插件体系

当前插件体系已经比较完整，不是简单预留目录：

- `src/zentex/core/plugin_base.py`
  - 定义插件基础契约、生命周期状态、回退与撤销约束

- `src/zentex/common/plugin_registry.py`
  - 负责统一插件注册、绑定、筛选、状态管理

- `src/zentex/core/plugin_runtime.py`
  - 定义插件运行态、健康检查、回退决策、撤销记录

- `src/plugins`
  - 已按功能拆分为：
    - `nine_questions`
    - `model_providers`
    - `cognitive`
    - `execution`
    - `sensory`
    - `simulation`
    - `memory`
    - `weights`
    - `reflection`

说明当前系统已经具备：

- 插件化接入
- 插件生命周期治理
- 健康/降级/回退约束
- 九问插件与能力补丁插件并存的扩展模型

### 2.4 LLM / Provider 接入

从 `provider_tools.py`、`model_provider_spec.py`、`web_console/services/llm.py` 以及全局 LLM guard 逻辑看，当前已实现：

- 默认 LLM Provider 工具装配
- Provider 配置检查
- API Key 缺失时 fail-closed
- LLM 调用异常分类映射
  - 认证错误
  - 配置错误
  - 限流
  - 超时
  - 返回结构错误
- Web Console 在写操作前进行 LLM 可用性拦截

这说明项目已经明确要求“真实 LLM 可用才允许进入相关主链”，而不是伪造 AI 结果。
